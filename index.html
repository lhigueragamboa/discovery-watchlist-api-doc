    Introduction

    The Watchlist API allows registered end consumers to save catalog assets to be watched at a later time. Watchlist items are saved per associated consumer, each of which have just one watch list known as the default watch list.

    There are two versions of this service, v1 and v2. The version to use is specified in the URL as shown in the examples that follow.

    You can view examples of calls to the API in the area to the right.

    Authentication

    curl 'http://player.ooyala.com/watchlist/<api_version>/list?account_token=<account_token>'

    Be sure to replace account_token with the one provided by the Gigya authentication service, and replace api_version with either v1 or v2.

    To start using the Watchlist service, obtain the API key or authentication token from your Ooyala representative.

    The Watchlist API requires that the account_token URL parameter be included in all requests.

    Watchlists

    Get The Default Watchlist

    curl 'http://player.ooyala.com/watchlist/<api_version>/list?account_token=<account_token>'

    The above command returns JSON structured like this:

    {
  "_shards": {
    "failed": 0,
    "successful": 4,
    "total": 4
  },
  "hits": {
    "hits": [],
    "max_score": 0.053994928,
    "total": 3
  },
  "timed_out": false,
  "took": 3
}

    This endpoint retrieves the metadata asociated with the IDs saved in the default watchlist.

    HTTP Request

    GET http://player.ooyala.com/watchlist/:api_version/list

    Query Parameters

    Parameter Description Type Required
    account_token Authentication token provided by Gigya String Yes
    api_version The version of the api the query is directed to. Must be either v1 OR v2 String Yes
    All requests to the Watchlist API must be authenticated.
    The internal hits field in the response has a complicated structure that is excluded from the examples.

    Error Codes

    Status Code Error Description
    400 Bad Request Empty URL parameter <name>
    500 Internal Server Error Parameters map is nil
    500 Internal Server Error Timed out retrieving metadata for watchlist items

    Add an Item

    curl -XPOST -H 'Content-Type: application/json' 'http://player.ooyala.com/watchlist/<api_version>/list?embed_code=9pMGJ4cDowP-bX8-aITZyNyuHH7dyeOk&account_token=<account_token>' -d '{}'

    The above command returns JSON structured like this:

    {
  "item_id": "9pMGJ4cDowP-bX8-aITZyNyuHH7dyeOk",
  "position": 1,
  "created_at": "2015-06-03T11:59:31.91646667-05:00",
  "updated_at": "2015-06-03T11:59:31.91646667-05:00"
}

    This endpoint adds an item to the default watchlist.

    HTTP Request

    POST http://player.ooyala.com/watchlist/:api_version/list

    URL Parameters

    Parameter Description Type Required
    account_token Authentication token provided by Gigya String Yes
    api_version The version of the api the query is directed to. Must be either v1 OR v2 String Yes
    embed_code The id of the item to be added to the default watch list String Yes

    Delete an Item

    curl -XDELETE 'http://player.ooyala.com/watchlist/<api_version>/list?embed_code=9pMGJ4cDowP-bX8-aITZyNyuHH7dyeOk&account_token=<account_token>'

    The above command returns JSON structured like this:

    {
  "success": true
}

    This endpoint deletes an item from the default watchlist.

    HTTP Request

    DELETE http://player.ooyala.com/watchlist/:api_version/list

    URL Parameters

    Parameter Description Type Required
    account_token Authentication token provided by Gigya String Yes
    api_version The version of the api the query is directed to. Must be either v1 OR v2 String Yes
    embed_code The id of the item to be added to the default watch list String Yes

    Error Codes

    Status Code Error Description
    400 Bad Request Empty URL parameter <name>
    500 Internal Server Error Parameters map is nil

    Check Whether an Item Exists in the Default Watchlist

    curl 'http://player.ooyala.com/watchlist/<api_version>/list/has?embed_code=9pMGJ4cDowP-bX8-aITZyNyuHH7dyeOk&account_token=<account_token>'

    The above command returns JSON structured like this:

    {
  "response": false
}

    This endpoint checks if an item exists in the default watchlist.

    HTTP Request

    GET http://player.ooyala.com/watchlist/:api_version/list/has

    URL Parameters

    Parameter Description Type Required
    account_token Authentication token provided by Gigya String Yes
    api_version The version of the api the query is directed to. Must be either v1 OR v2 String Yes
    embed_code The id of the item to be added to the default watch list String Yes

    Error Codes

    Status Code Error Description
    400 Bad Request Empty URL parameter <name>
    500 Internal Server Error Parameters map is nil

    Clear the default Watchlist

    curl -XDELETE 'http://player.ooyala.com/watchlist/<api_version>/list/all?account_token=<account_token>'

    The above command returns JSON structured like this:

    {
  "sucess": false
}

    This endpoint deletes all the items in the default watchlist.

    HTTP Request

    DELETE http://player.ooyala.com/watchlist/:api_version/list/all

    URL Parameters

    Parameter Description Type Required
    account_token Authentication token provided by Gigya String True
    api_version The version of the api the query is directed to. Must be either v1 OR v2 String True

    Error Codes

    If an error occurs, a 500 status code is returned along with the following JSON body:

    {
  "success": false
}
